## 1. 산업·고용 — 사업체수 (★§8.6 숙제 해결)

- **표**: `DT_1BD1032`(기업 수)·`DT_1BD1035`(종사자 수). org 101, 년/2023.
- **차원**: 지역(census 11190) × 산업대분류(A, 전체=`0`) × 항목(T01활동·T02신생·T03소멸).
- **②축**: 산업·고용 → **use_types: 상업**(배후 경제활동 규모).
- **headline 슬라이스**: 전산업 활동 사업체수 — 영등포 **96,993개** (2023).
- **③파생**:
  - 사업체밀도(인구당) = 사업체수 / 총인구 × 1,000 = 96,993/371,362×1,000 = **261개/천명**.
  - 신생률 = 신생/활동 = 10,698/96,993 = **11.0%** (상권 역동성). 소멸률 동일 방식.
  - (면적 확보 시) 사업체밀도/㎢.
- **④교차** (산업대분류별 objL2=ALL) — 영등포 동네 산업구조 프로파일(2023):
  부동산업 23,490 · 도소매 17,746 · 건설 11,141 · 숙박음식 7,950 · 금융보험 7,542 · 전문과학기술 5,656 …
  → "여의도=부동산·금융 집적" 같은 **동네 성격**이 드러남. 단일 숫자 못 주는 것.
- **출처·체계**: KOSIS 전국사업체조사, reg≠census — **지역코드 census(11190)**. 기준 2023.
- **편입메모**: `direct`+`objL2_pick="0"`(전산업 합계)로 사업체수 1지표. 산업구성은 별도 프로파일 필드(교차).

## 2. 주거 — 빈집 (노후·공실)

- **차원**: 지역 × 주택종류(전체=`00`) × 건축연도.
- **②축**: 주거 → **use_types: 주거**(공실·노후 리스크).
- **headline**: 빈집 계 — 영등포 **2,292호** (2024).

- **④교차** (건축연도별): 2005~09: 81 · 2000~04: 184 · 1990~99: 331 … → **노후 빈집 분포**(오래된 빈집일수록 정비·재생 시그널).

## 요약 — 편입 난이도 분류 (Phase 3 로드맵)

| 난이도 | 표 | 편입 방법 |
| --- | --- | --- |
| 쉬움(순수 JSON) | 신혼부부·주택거래(단순 2차원) | matrix.json + objL1=프로파일코드 |
| 중간(objL2 합계) | 사업체수·빈집·등록장애인 | objL2_pick=합계코드. **fetch_table objL2 이미 지원** → 대부분 JSON |
| 어려움(objL3+·롱코드) | 생애단계(2분류)·의료인력 | objL3 지원 위해 fetch_table 확장(코드) 필요 |

**핵심**: ①크랙으로 *전부 열림*. 중간난이도(사업체수·빈집·장애인)는 기존 `objL2_pick` 패턴(총인구수가 쓰는)으로
**코드 변경 없이 편입 가능** — census 지역코드만 `region_scheme:"census"`로 표기(이미 1인가구비율이 쓰는 방식).
objL3+ 다분류·롱코드 표만 추후 fetch_table 확장 대상.
